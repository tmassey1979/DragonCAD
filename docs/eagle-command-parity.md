# EAGLE Command Parity Matrix

This matrix tracks common EAGLE commands and tools against the current DragonCAD direction. Status values are intentionally limited to `Implemented`, `Partial`, `Planned`, and `Intentionally different` so the table can be validated.

Use this as migration guidance, not as a promise that DragonCAD will preserve EAGLE's command syntax exactly.

| EAGLE command/tool | DragonCAD equivalent | Status | Notes | Follow-up |
| --- | --- | --- | --- | --- |
| OPEN project/design | Local workspace and project folder flows | Partial | DragonCAD has local project-folder foundations, but the project center and full shell open flow are still being rounded out. | [PRJ-001 issue](https://github.com/tmassey1979/DragonCAD/issues/13) |
| SAVE | DragonCAD project persistence and shell save/open commands | Partial | Core project folder persistence exists, while shell save/open commands remain a focused story. | [PRJ-003 issue](https://github.com/tmassey1979/DragonCAD/issues/15) |
| ADD | Component placement from trusted DragonCAD libraries | Partial | Placement and deterministic references exist, but chooser repeat flow and component-editor creation tools still need polish. | [Editor backlog: EDIT-003](editor-interaction-backlog.md#edit-003---component-placement-repeat-flow) |
| MOVE | Schematic component movement and board component movement | Implemented | DragonCAD supports moving placed schematic components and board components through editor view-model commands. | Current editor behavior |
| DELETE | Delete selected schematic or board items | Implemented | DragonCAD exposes deletion for selected schematic and board entities that are already modeled by the editor. | Current editor behavior |
| ROTATE | Rotate schematic and board components | Implemented | Rotation is available for placed components in the current editor surfaces. | Current editor behavior |
| MIRROR | Mirror schematic and board components | Implemented | Mirroring is available for placed components where the current editor model supports it. | Current editor behavior |
| NET / WIRE | Schematic pin-to-wire routing and net creation | Partial | Pin-to-wire routing exists, while visible net labels and richer net editing remain planned. | [SCH-002 issue](https://github.com/tmassey1979/DragonCAD/issues/6) |
| LABEL | Schematic net label editing and rendering | Planned | Net label editing/rendering is planned as a schematic editor follow-up. | [SCH-002 issue](https://github.com/tmassey1979/DragonCAD/issues/6) |
| NAME | Component reference and net naming workflows | Partial | Component reference editing exists in current editor state; richer net naming and label association are planned. | [SCH-002 issue](https://github.com/tmassey1979/DragonCAD/issues/6) |
| VALUE | Component value editing | Partial | Placed component value editing exists, but broader attribute/package editing is not complete. | [Editor backlog: EDIT-004](editor-interaction-backlog.md#edit-004---component-properties-editing) |
| SMASH | Separate text placement for names and values | Planned | DragonCAD intends explicit text/label editing instead of EAGLE's smash behavior. | [DOC-007 issue](https://github.com/tmassey1979/DragonCAD/issues/32) |
| INVOKE | Gate or alternate symbol-unit placement | Planned | Multi-unit symbol behavior belongs with richer symbol/component editor fidelity. | [DOC-007 issue](https://github.com/tmassey1979/DragonCAD/issues/32), [Editor backlog: EDIT-005](editor-interaction-backlog.md#edit-005---symbol-geometry-fidelity) |
| GROUP | Multi-select and batch editor operations | Planned | Group selection is not documented as a completed editor behavior. | [DOC-006 issue](https://github.com/tmassey1979/DragonCAD/issues/31) |
| COPY | Duplicate selected editor entities | Planned | Copy/duplicate commands should be documented with the command catalog once tool semantics are stable. | [DOC-006 issue](https://github.com/tmassey1979/DragonCAD/issues/31) |
| PASTE | Paste copied editor entities | Planned | Paste behavior should follow DragonCAD command semantics rather than raw EAGLE command parsing. | [DOC-006 issue](https://github.com/tmassey1979/DragonCAD/issues/31) |
| ROUTE | Manual PCB routing | Partial | DragonCAD creates manual orthogonal traces; pad start/finish, airwire retirement, and 45-degree mode remain open. | [PCB-001 issue](https://github.com/tmassey1979/DragonCAD/issues/9) |
| RIPUP | Remove routed copper / restore connection review | Partial | Trace deletion exists; complete airwire restoration and route-state semantics are still planned. | [PCB-002 issue](https://github.com/tmassey1979/DragonCAD/issues/10) |
| VIA | Insert via while routing | Partial | Vias exist as board primitives, while route-time via insertion and richer via editing remain planned. | [Editor backlog: Wave 3 via insertion](editor-interaction-backlog.md#wave-3-plan-and-status) |
| CHANGE WIDTH | Trace width editing | Planned | Trace width editing is planned as a board editor workflow. | [DOC-008 issue](https://github.com/tmassey1979/DragonCAD/issues/33), [Editor backlog: Wave 4 trace width editing](editor-interaction-backlog.md#wave-4-plan-and-status) |
| CHANGE LAYER | Active layer and trace layer changes | Partial | Layer selection, visibility, and trace layer reassignment exist; layer palette polish remains. | [Editor backlog: EDIT-008](editor-interaction-backlog.md#edit-008---layer-palette-and-visibility-controls) |
| DISPLAY | Layer visibility controls | Partial | Board layer visibility exists, with broader editor layer-control polish remaining. | [Editor backlog: EDIT-008](editor-interaction-backlog.md#edit-008---layer-palette-and-visibility-controls) |
| GRID | Grid visibility and snap spacing | Partial | Grid visibility/style/spacing commands exist; persistence and unit polish remain. | [Editor backlog: EDIT-009](editor-interaction-backlog.md#edit-009---grid-controls-and-snap-modes) |
| ERC | Schematic electrical rule diagnostics | Implemented | DragonCAD has a schematic electrical rule checker model and tests. | [SCH-004 issue](https://github.com/tmassey1979/DragonCAD/issues/8) |
| DRC | Board design-rule checks | Planned | Board-level DRC should be implemented as a dedicated validation workflow after routing and fabrication primitives stabilize. | [DOC-008 issue](https://github.com/tmassey1979/DragonCAD/issues/33) |
| CAM processor | DragonCAD fabrication output and handoff review | Partial | Gerber/drill manifests, BOM, pick-and-place, and handoff foundations exist; live provider upload remains deferred. | [Fabrication roadmap](remaining-implementation-roadmap.md#epic-c---fabrication) |
| EXPORT BOM | BOM CSV and sourcing review exports | Partial | BOM export foundations exist, while richer BOM order review and marketplace handoff remain review-first roadmap items. | [Marketplace roadmap: CMP-004](component-marketplace-roadmap.md#cmp-004---bom-order-review) |
| PRINT / image export | Documentation and release artifact workflow | Planned | Public release notes and polished documentation artifacts are tracked separately from editor behavior. | [DOC-008 issue](https://github.com/tmassey1979/DragonCAD/issues/33) |
| RUN ULP | DragonCAD typed commands, reports, plugins, and review workflows | Intentionally different | DragonCAD does not execute arbitrary EAGLE ULP scripts; migrate ULP intent into explicit DragonCAD commands or future plugins. | [Migration guide: ULP migration direction](eagle-migration.md#ulp-migration-direction) |
| SCRIPT | DragonCAD command catalog and future automation/plugin surfaces | Intentionally different | DragonCAD should prefer typed, testable commands over replaying EAGLE script side effects. | [DOC-006 issue](https://github.com/tmassey1979/DragonCAD/issues/31) |
| AUTOROUTER | Manual routing first; autorouting deferred | Intentionally different | Autorouting is not part of the current DragonCAD routing scope. | [Migration guide: Known limitations](eagle-migration.md#known-limitations) |
| LIBRARY editor | Component editor and trusted-library promotion | Partial | Component previews and package options exist; dedicated creation/editing tools and reviewed promotion remain ongoing. | [CMP-004 issue](https://github.com/tmassey1979/DragonCAD/issues/4) |
| Package/device mapping | DragonCAD component definitions and package choices | Partial | Package preview and component metadata exist, with fuller package switching/filtering and component editor polish planned. | [Editor backlog: EDIT-010](editor-interaction-backlog.md#edit-010---component-editor-creation-tools) |
| Import `.sch` | EAGLE schematic import input and project assembly planning | Partial | `.sch` is accepted by the import assembly planner and paired with same-name board/library files; downstream fidelity remains review-required. | [PRJ-004 issue](https://github.com/tmassey1979/DragonCAD/issues/16) |
| Import `.brd` | EAGLE board import input and project assembly planning | Partial | `.brd` is accepted by the import assembly planner and paired with same-name schematic/library files; board fidelity remains review-required. | [PRJ-004 issue](https://github.com/tmassey1979/DragonCAD/issues/16) |
| Import `.lbr` | EAGLE library import input and candidate-library review | Partial | `.lbr` is accepted by the import assembly planner, but imported library content should remain review-required until promoted. | [PRJ-004 issue](https://github.com/tmassey1979/DragonCAD/issues/16) |
