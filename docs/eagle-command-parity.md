# EAGLE Command Parity Matrix

This matrix tracks the EAGLE workflows that migrating users expect to find in DragonCAD. Status values are intentionally limited to `Implemented`, `Partial`, `Planned`, and `Intentionally different` so the documentation validator can enforce the completion contract.

Use this as migration guidance, not as a promise that DragonCAD will preserve EAGLE's command syntax exactly. DragonCAD command ids come from the command catalog when a typed command already exists; `Not available` means the current behavior is direct editor interaction, pending command work, or an intentional non-command replacement.

| EAGLE command/tool | DragonCAD workflow | Status | DragonCAD command id | Help topic | Implementation issues | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| ADD | Place a trusted component from the component catalog into the schematic. | Implemented | `PlaceSelectedComponentCommand`, `PlaceArmedComponentOnSchematicCommand` | [Command catalog](help/reference/command-catalog.md) | [EDIT-028](https://github.com/tmassey1979/DragonCAD/issues/84), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | DragonCAD preserves the placement intent while making the selected library source and armed placement state visible. |
| MOVE | Select and move schematic parts, board footprints, traces, and vias on the active editor surface. | Partial | `ActivateSelectToolCommand`, `ActivateBoardSelectToolCommand` | [Editor acceptance checklist](ux/editor-acceptance-checklist.md) | [EDIT-038](https://github.com/tmassey1979/DragonCAD/issues/94), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | Movement exists through selection tools; visual handles and hover feedback remain an open polish story. |
| COPY | Duplicate selected schematic objects and use group copy/paste where available. | Partial | `DuplicateSelectedPartCommand` | [Command catalog](help/reference/command-catalog.md) | [EDIT-033](https://github.com/tmassey1979/DragonCAD/issues/89), [EDIT-034](https://github.com/tmassey1979/DragonCAD/issues/90) | Single-part duplication is cataloged; schematic group copy/paste remains open and PCB group copy/paste is tracked separately. |
| DELETE | Delete the active schematic or board selection. | Implemented | `DeleteActiveSelectionCommand`, `DeleteBoardSelectionCommand` | [Command catalog](help/reference/command-catalog.md) | [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31), [SHELL-012](https://github.com/tmassey1979/DragonCAD/issues/101) | DragonCAD exposes delete from the shared command surface for modeled editor selections. |
| GROUP | Multi-select objects and apply batch editor operations. | Planned | Not available | [Eagle-Plus standards](ux/eagle-plus-standards.md) | [EDIT-033](https://github.com/tmassey1979/DragonCAD/issues/89), [EDIT-034](https://github.com/tmassey1979/DragonCAD/issues/90) | Planned parity depends on schematic and PCB group command completion. |
| ROUTE | Route PCB traces manually on the active layer and finish the route explicitly. | Implemented | `ActivateBoardRouteToolCommand`, `FinishBoardRouteCommand` | [Command catalog](help/reference/command-catalog.md) | [PCB-001](https://github.com/tmassey1979/DragonCAD/issues/9), [EDIT-025](https://github.com/tmassey1979/DragonCAD/issues/75), [EDIT-027](https://github.com/tmassey1979/DragonCAD/issues/80) | DragonCAD covers manual route start, pad-based completion, airwire retirement, and forty-five degree routing follow-up. |
| RIPUP | Remove routed copper and restore connection review where route state supports it. | Partial | `RipupSelectedTraceCommand` | [Command catalog](help/reference/command-catalog.md) | [PCB-002](https://github.com/tmassey1979/DragonCAD/issues/10), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | Trace deletion exists; the command catalog still marks dedicated ripup command parity as missing. |
| VIA | Place a via during board work or insert one into a selected trace segment. | Implemented | `PlaceBoardViaCommand`, `InsertBoardViaIntoSelectedTraceSegmentCommand` | [Command catalog](help/reference/command-catalog.md) | [EDIT-023](https://github.com/tmassey1979/DragonCAD/issues/70), [EDIT-016](https://github.com/tmassey1979/DragonCAD/issues/60) | DragonCAD supports via placement, trace insertion, and via size editing as board primitives. |
| CHANGE | Change layer, package, value, and related object attributes through explicit property workflows. | Partial | `MoveSelectedBoardTraceToLayerCommand` | [Editor acceptance checklist](ux/editor-acceptance-checklist.md) | [EDIT-029](https://github.com/tmassey1979/DragonCAD/issues/85), [EDIT-030](https://github.com/tmassey1979/DragonCAD/issues/86), [EDIT-036](https://github.com/tmassey1979/DragonCAD/issues/92) | Layer change has a command id; package and attribute changes are implemented as editor/property workflows rather than one EAGLE-style umbrella command. |
| DISPLAY | Toggle layer visibility and inspect active board layer state. | Implemented | `ToggleSelectedBoardLayerVisibilityCommand`, `SetActiveBoardLayerCommand`, `OpenLayerSettingsCommand` | [Command catalog](help/reference/command-catalog.md) | [EDIT-036](https://github.com/tmassey1979/DragonCAD/issues/92), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | Core board layer visibility is command-cataloged; complete layer settings remain represented by the missing catalog command. |
| GRID | Show, hide, and configure grid and snap behavior. | Partial | `SetGridSizeCommand`, `ToggleGridSnapCommand`, `ToggleGridVisibilityCommand` | [Command catalog](help/reference/command-catalog.md) | [EDIT-035](https://github.com/tmassey1979/DragonCAD/issues/91), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | Grid visibility is exposed, while schematic layer palette and grid persistence remain open. |
| SMASH | Separate name/value text placement into explicit text and label editing behavior. | Implemented | Not available | [Eagle-Plus standards](ux/eagle-plus-standards.md) | [EDIT-031](https://github.com/tmassey1979/DragonCAD/issues/87), [DOC-007](https://github.com/tmassey1979/DragonCAD/issues/32) | DragonCAD modernizes EAGLE smash behavior as explicit text and label placement rather than preserving the exact command. |
| INVOKE | Place alternate gates or symbol units for multi-unit parts. | Planned | Not available | [EAGLE migration guide](eagle-migration.md) | [EDIT-032](https://github.com/tmassey1979/DragonCAD/issues/88), [DOC-007](https://github.com/tmassey1979/DragonCAD/issues/32) | Multi-unit gate invoke placement is an open roadmap story. |
| NAME | Edit component references and net names with visible property or label workflows. | Partial | `AddNetLabelCommand` | [Command catalog](help/reference/command-catalog.md) | [SCH-002](https://github.com/tmassey1979/DragonCAD/issues/6), [EDIT-026](https://github.com/tmassey1979/DragonCAD/issues/79), [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31) | Net label behavior exists in the roadmap and catalog, but the catalog still marks visible label command parity as missing. |
| VALUE | Edit component value, package, and attribute data through component properties. | Partial | Not available | [Editor acceptance checklist](ux/editor-acceptance-checklist.md) | [EDIT-029](https://github.com/tmassey1979/DragonCAD/issues/85), [CMP-011](https://github.com/tmassey1979/DragonCAD/issues/103) | Value and attribute editing is represented by editor workflows rather than a standalone command id. |
| ERC | Run schematic electrical rule diagnostics. | Implemented | Not available | [EAGLE migration guide](eagle-migration.md) | [SCH-004](https://github.com/tmassey1979/DragonCAD/issues/8), [DOC-011](https://github.com/tmassey1979/DragonCAD/issues/125) | The schematic ERC diagnostics model is implemented; a catalog command id has not been assigned. |
| DRC | Review board design rules, fabrication readiness, and order blockers before handoff. | Implemented | `ReviewFabricationOrderCommand` | [Command catalog](help/reference/command-catalog.md) | [PCB-006](https://github.com/tmassey1979/DragonCAD/issues/96), [MKT-039](https://github.com/tmassey1979/DragonCAD/issues/110) | DragonCAD treats DRC as review-first validation tied to fabrication readiness. |
| EXPORT | Export manufacturing packages, BOM CSV, and reviewed handoff artifacts. | Implemented | `ExportManufacturingPackageCommand`, `PrepareMarketplaceBomCsvCommand` | [Command catalog](help/reference/command-catalog.md) | [FAB-009](https://github.com/tmassey1979/DragonCAD/issues/112), [FAB-007](https://github.com/tmassey1979/DragonCAD/issues/72), [MFG-002](https://github.com/tmassey1979/DragonCAD/issues/18) | EAGLE CAM/export intent maps to deterministic DragonCAD manufacturing and BOM outputs. |
| SCRIPT/ULP | Replace arbitrary EAGLE script and ULP execution with typed commands, reports, and future reviewed plugins. | Intentionally different | Not available | [EAGLE migration guide](eagle-migration.md#ulp-migration-direction) | [DOC-006](https://github.com/tmassey1979/DragonCAD/issues/31), [DOC-007](https://github.com/tmassey1979/DragonCAD/issues/32) | DragonCAD does not execute arbitrary EAGLE automation; migration should preserve intent through auditable command surfaces. |
| LIBRARY | Create, edit, validate, import, and promote trusted component library content. | Partial | `NewComponentEditorCommand`, `OpenSelectedComponentEditorCommand`, `AddComponentEditorStarterGeometryCommand`, `ValidateComponentDraftCommand` | [Command catalog](help/reference/command-catalog.md) | [CMP-011](https://github.com/tmassey1979/DragonCAD/issues/103), [CMP-012](https://github.com/tmassey1979/DragonCAD/issues/104), [CMP-014](https://github.com/tmassey1979/DragonCAD/issues/106), [PRJ-004](https://github.com/tmassey1979/DragonCAD/issues/16) | Component editor and trusted-library persistence exist; datasheet promotion command UI and import review workflows continue separately. |
