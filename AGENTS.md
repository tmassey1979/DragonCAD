# DragonCAD Agent Rules

## UI Screenshot And Wiki Header Rule

When a pass changes visible UI, shell layout, editor canvas behavior, toolbar/menu presentation, theme, or any screen that appears in the GitHub Pages/wiki documentation:

- Run the app after the change and capture a fresh screenshot of the changed screen.
- Update the relevant page header image with the new screenshot before closing the pass.
- If multiple UI screens changed, update the most relevant page header image for each affected documentation/wiki page.
- Mention the screenshot/header update in the issue or final status comment.
- If screenshot capture is blocked, document the blocker and do not claim the UI documentation is current.
